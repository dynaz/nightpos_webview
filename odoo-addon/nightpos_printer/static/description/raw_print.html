1. Main POS receipt flow

When nightpos_receipt_mode is tcp or sunmi and the POS runs inside the NightPOS Android WebView, receipts use the same layout as Odoo (OrderReceipt), not custom plain text.

pos.order + session config ↓ PosStore.printReceipt() point_of_sale …/pos_store.js ↓ PrinterService.print(OrderReceipt, …) printer_service.js ↓ BasePrinter.printReceipt(html) base_printer.js → htmlToCanvas → JPEG base64 ↓ NightposEscposPrinter.sendPrintingJob(imageBase64) static/src/pos/nightpos_escpos_printer.js ↓ RPC print_image(image, printer_id) controllers/main.py ↓ NightposPrinter._image_to_escpos(image_b64) models/nightpos_printer.py → escpos_b64 ↓ NightposEscposPrinter._sendRaw(escpos_b64, printer) ↓ TcpPrinter.printRaw or SunmiPrinter.printRaw (Flutter bridge)
Wiring: pos_receipt_patch.js registers NightposEscposPrinter in afterProcessServerData() via this.printer.setPrinter(…), then printReceipt() calls super.printReceipt() so Odoo renders the receipt.

2. What data is printed

Receipt content (not from nightpos.printer)

  • Model: pos.order (current order in POS)
  • Template: point_of_sale.OrderReceipt — header, lines, taxes, payments, change, footer
  • Rendered by: Odoo renderer / htmlToCanvas (same as IoT / ePOS printers)

Printer connection (from nightpos.printer)

  • pos.config.nightpos_receipt_printer_id → nightpos.printer
  • Fields used at print time: id, ip, port, name (via _as_dict())
  • Sunmi mode does not need IP; TCP mode sends ip + port to TcpPrinter

6. Other print paths

Test print (backend)

action_test_print() → _generate_test_escpos() → client action nightpos_printer.test_print → nightpos_test_print_action.js → printRaw().

Kitchen / preparation

Not printed directly by this module. pos_config._sync_nightpos_preparation_printers() mirrors nightpos.printer to pos.printer; Odoo POS prints preparation tickets using standard POS printer logic.

7. Configuration fields

ip, port nightpos.printer TCP target for TcpPrinter.printRaw

8. Server dependencies

Image receipt printing requires on the Odoo server: pip install pillow python-escpos (declared in module external_dependencies).

Without these packages, print_image / _image_to_escpos will return an error to the POS.
